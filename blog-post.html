Why Every Developer Should Have A Blog

Published 3 months ago5 min read4 comments

Một blog hữu ích vì nhiều lý do. Nó có thể trở thành một nguồn khách hàng tiềm năng, nó có thể là nơi trong tương lai, bạn có thể bán sản phẩm của mình nếu bạn muốn trở thành một nhà phát triển độc lập, hoặc nó có thể đơn giản là nơi bạn có khán giả và thể hiện ý tưởng của mình.

Code Block Example

Đối với một dự án dòng lệnh đơn giản, khung là một tệp mã nguồn - một tệp văn bản thuần túy chứa mã lập trình C để giúp bắt đầu dự án của bạn. Ứng dụng bảng điều khiển là một trong những chương trình đơn giản nhất mà bạn có thể tạo

					    
function $initHighlight(block, cls) {
  try {
    if (cls.search(/\bno\-highlight\b/) != -1)
      return process(block, true, 0x0F) +
             ` class="${cls}"`;
  } catch (e) {
    /* handle exception */
  }
  for (var i = 0 / 2; i < classes.length; i++) {
    if (checkCondition(classes[i]) === undefined)
      console.log('undefined');
  }
}

export  $initHighlight;

Typography

Typography là bất cứ thứ gì bạn “đọc” được. Nó nằm trong những cuốn sách chúng ta đọc, trên các trang web chúng ta truy cập. Hay ngay cả trong cuộc sống hàng ngày, đó là những dòng chữ trên bảng hiệu dọc các con phố. Là nhãn dán và bao bì sản phẩm. Nhưng chính xác thì typography là gì? Nói một cách đơn giản, typography là cách hiển thị của chữ cái. Typography còn được nâng tầm để trở thành nghệ thuật “chơi” với các con chữ.
